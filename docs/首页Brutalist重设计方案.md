# 首页 Brutalist Minimalism 重设计方案

## 设计背景

当前首页偏向"友好极简"——白底、圆角、柔和的 zinc 色背景、渐变式动画。视觉上干净但缺乏张力和记忆点。

Brutalist 核心理念：暴露结构、拒绝装饰、Typography 即设计、对比即力量。

已有的 JetBrains Mono 等宽字体是天然的 Brutalist 基因，交互式展开文本的机制也非常适合——用户"解码"一个系统，这本身就很 Brutal。

---

## 第一轮方案（已淘汰）

### 方向 A-v1：「TERMINAL SYS」系统终端

纯黑底 + 白字 + 方括号交互词 + 终端状态栏。已实现但效果像"黑板"，缺乏深度和氛围感。

**问题诊断：**

1. 纯平黑底 + 白字 = 暗色模式，不是终端。真正的 CRT 字符是**发光**的
2. 没有深度/纹理 — 纯色 `#0a0a0a` 没有层次感
3. 终端隐喻只停留在表面 — 方括号 + 状态栏只是装饰
4. `#00FF66` 绿色用得太克制，没有形成统一氛围

### 方向 B-v1：「EDITORIAL BRUTAL」粗野编辑（备选）

米白底 `#F0EDE8` + 纯黑文字 + 红色强调 + 超大竖排名字 + 不对称布局。偏杂志/艺术方向。

## 第二轮方案

### 方向 A：「PHOSPHOR CRT」磷光显示器（已选，与 C 混合）

**核心概念：** 整个页面像一台老式磷光 CRT 显示器。文字不是"印"在黑色上的——而是**发光**的。

**视觉关键词：** VT100 / Apple II / IBM 5151 / 磷光余辉

```
视觉效果：

┌─ 画面四周有暗角（vignette）──────────────────┐
│                                                │
│  ░░░ 极细的水平扫描线贯穿全屏 ░░░              │
│                                                │
│  KLAUDEN_     ← 绿色文字带 glow（text-shadow） │
│                                                │
│  你好，我是 Klauden，来自 [中国] 的             │
│  [全栈设计师]...                                │
│     ↑                                          │
│     可点击词有更强的发光效果                     │
│                                                │
│  ▊  ← 闪烁的块状光标                           │
│                                                │
│  ──────────────────────────────────────         │
│  ● 3/7 QUERIES │ SYS.ACTIVE │ 2025.02          │
│                                                │
└── 底部边缘也有暗角 ─────────────────────────────┘
```

**设计规范：**

- **背景：** 深黑 `#050505`，不是纯平色——带极细扫描线 + 微弱噪点
- **文字发光：** `text-shadow: 0 0 8px rgba(0,255,102,0.6)` 让文字像磷光一样发光
- **扫描线：** CSS `repeating-linear-gradient` 水平线，极低透明度
- **暗角：** `radial-gradient` 模拟 CRT 边缘变暗
- **噪点：** CSS/SVG noise 纹理叠加，极微弱的静态噪声
- **光标：** 块状光标 `█` 用 CSS animation 闪烁
- **配色：** 深黑底 `#050505` + 磷光绿 `#00FF66`（主色）+ 白色仅用于高亮
- **可选：** 页面加载时的 "boot sequence"（1-2秒启动序列）

**记忆点：** "这个网站的文字在发光" — 做过或见过 CRT 显示器的人会立刻共鸣。

### 方向 B：「NEOVIM RICE」代码编辑器

**核心概念：** 整个首页像一个精心配置的 Neovim / VS Code。不同类型的内容用不同的"语法高亮"颜色。

**视觉关键词：** Tokyo Night / Catppuccin / Dracula / "dotfiles porn"

```
视觉效果：

╭── tabs ──────────────────────────────────────╮
│ 📄 home.tsx  │  craft.tsx  │  projects.tsx    │
├── line numbers ──────────────────────────────┤
│  1  // KLAUDEN — Full Stack Designer          │
│  2  // Shanghai, China                        │
│  3                                            │
│  4  你好，我是 Klauden，                       │
│  5  来自 中国 的 全栈设计师                     │
│     ↑紫色    ↑青色(keyword)                    │
│  6  如果您有 合作需求，                         │
│              ↑黄色(string-like)                │
│  7  欢迎通过 邮箱 联系                         │
│              ↑蓝色(link)                       │
│  8                                            │
├───────────────────────────────────────────────┤
│  NORMAL │ home.tsx │ utf-8 │ 3/7 │ ln 5 col 8│
╰───────────────────────────────────────────────╯
```

**设计规范：**

- **配色：** Tokyo Night — 深蓝底 `#1a1b26` + 紫 `#bb9af7` + 青 `#7dcfff` + 绿 `#9ece6a` + 橙 `#ff9e64`
- **行号：** CSS `counter-reset` / `counter-increment`
- **Tab bar：** 对应 Nav 页面
- **语法高亮：** 不同内容类型用不同颜色区分
- **状态栏：** 模仿 vim statusline（模式 + 文件名 + 编码 + 位置）

**记忆点：** "这个人的个人网站就是一个 Neovim" — 对程序员极具共鸣。

### 方向 C：「SYS.ACCESS」系统入侵 / 赛博HUD（已选，与 A 混合）

**核心概念：** 你打开了一个受限系统的访问终端。有实时数据在流动，有扫描线在运行，有数据面板在更新。

**视觉关键词：** Mr. Robot / Watch Dogs / Cyberpunk HUD / NMAP

```
视觉效果：

┌──────────────────────────────────────────────┐
│ ┌─ SYS ─┐                    ┌─ STATUS ────┐│
│ │ ▓▒░   │  KLAUDEN_          │ ACTIVE      ││
│ │ CPU 23%│                    │ PID 1337    ││
│ │ MEM 4G │                    │ UPTIME 847d ││
│ └────────┘                    └─────────────┘│
│                                              │
│  $ cat profile.txt                           │
│  你好，我是 Klauden，来自 [中国] 的          │
│  [全栈设计师]...                              │
│                                              │
│  > 鼠标坐标: (342, 567)   时间: 14:23:05    │
│                                              │
│  ┌─ QUERY LOG ───────────────────────────┐   │
│  │ [14:22:01] CLICK → 中国              │   │
│  │ [14:22:15] CLICK → 全栈设计师         │   │
│  │ [14:23:01] CLICK → 合作需求           │   │
│  └───────────────────────────────────────┘   │
│                                              │
│  3/7 QUERIES COMPLETE ████████░░░░ 43%       │
└──────────────────────────────────────────────┘
```

**设计规范：**

- **面板：** ASCII box drawing characters (`┌─┐│└─┘`) 做面板边框
- **实时数据：** 装饰性更新元素（时间、鼠标坐标、"uptime"）
- **Query Log：** 记录每次交互的时间和目标
- **进度条：** `████░░░░` 字符渲染
- **背景：** 极淡的 grid 或 dot pattern
- **命令前缀：** `$ cat profile.txt` 式的命令提示
- **配色：** 深黑 `#0a0a0a` + 绿 `#00FF66`（主指令）+ 琥珀 `#FFB800`（系统数据）+ 红 `#FF3333`（警告/活跃）

**记忆点：** "我在入侵他的个人资料" — 每次点击都被"记录"在 query log 里。

## 三个方向对比

| 维度 | A: Phosphor CRT | B: Neovim Rice | C: SYS.ACCESS |
|------|-----------------|----------------|---------------|
| 氛围 | 怀旧、温暖、有质感 | 极客、精致、有归属感 | 戏剧化、紧张、沉浸 |
| 技术复杂度 | 中（主要是 CSS 效果） | 中（布局 + 多色系统） | 高（实时数据 + 多面板） |
| 独特性 | 高 — glow效果很少见 | 高 — 语法高亮做网站少见 | 中高 — HUD类有但做好的少 |
| 适合人群 | 所有人（复古美感通用） | 程序员/极客（强共鸣） | 偏年轻/游戏玩家群体 |
| 改动范围 | 在现有基础上加层效果 | 需要重构布局和色彩系统 | 需要新增多个面板组件 |
| 与其他页面的延展性 | 好（效果层通用） | 很好（editor 概念统一） | 一般（面板设计定制性强） |

---

## 最终选择：A + C 混合「PHOSPHOR CRT × SYS.ACCESS」

以 Phosphor CRT 的视觉氛围（发光文字、扫描线、暗角、噪点）作为基底，融合 SYS.ACCESS 的交互面板元素（query log、进度条、实时数据）。

---

## 共通的 Brutalist 改动（无论选哪个方向）

1. **全站零圆角** — 所有 `rounded-*` 改为 `rounded-none`
2. **去掉阴影** — 用 1px border 替代所有 `shadow-*`
3. **去掉 zinc 渐变背景** — 交互词用边框或反色表达
4. **字体统一 JetBrains Mono** — 去掉 Noto Sans/Serif
5. **光标改为 crosshair** — 交互元素
6. **打字机效果调整** — 更机械、更硬、带块状光标
